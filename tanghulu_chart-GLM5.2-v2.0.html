Encodes price (OHLC), volume profile and order-flow imbalance (Delta) in a single element. Each "tanghulu" stacks symmetric beads on a central axis: bead size = total volume at that price; left-red (buy) / right-green (sell) split reveals the imbalance via the divider offset; gold outline = POC; blue band = Value Area.

01 图形元素解码 Reading the Shape / 形状解读

  • 中心轴 Stick：垂直细线，连接当日最高价(High)与最低价(Low)。 Vertical line connecting High and Low.
  • 珠子 Bead：每个价格层一颗，左右对称，宽度∝该层总成交量。 One per price level, left-right symmetric, width ∝ total volume.
  • 分色 Split：珠子内左红=主买、右绿=主卖，分界线偏移=买卖失衡。 Left-red = buy, right-green = sell; divider offset = imbalance.
  • 金色描边 POC：当日总成交量最大的价格，市场"公允价值"。 Price with the highest total volume = fair value.
  • 蓝色背景带 Value Area：围绕POC累计约70%成交量的价格区间。 Price range holding ~70% of volume around POC.
  • 开/收标记 Open/Close：轴左侧短横=开盘，右侧短横=收盘，红涨绿跌。 Left tick = Open, right tick = Close; red=up, green=down.

02 如何读出市场结构 Interpreting Structure / 市场结构

  • 单日：POC=价值共识；珠子最大处=成交密集区。 POC = value consensus; largest bead = volume node.
  • 多日趋势：POC逐日抬高=多头主导；逐日下移=空头主导。 Rising POC = buyers in control; falling POC = sellers in control.
  • 失衡切换：同价位珠子红绿分界方向反转=多空控制权易手。 Divider flipping direction at a level = control handover.
  • 高位主卖堆积：创新高但顶部珠子绿色占比大=派发/顶部特征。 New high but green-dominant top beads = distribution.
  • 低位主买堆积：新低但底部红色占比大=吸筹/底部特征。 New low but red-dominant bottom beads = accumulation.
  • 价值区迁移：VA上移/下移=趋势；VA重叠=震荡。 Migrating VA = trend; overlapping VA = balance.

03 设计说明：Delta如何编码 How Imbalance Is Encoded / 失衡编码

  • 对称轮廓：每颗珠子左右等宽（宽度=该层总成交量），形状对称美观。 Symmetric outline: both halves equal width = total volume.
  • 比例分色：珠子内按主买/主卖占比，左红右绿竖直分割。 Proportional split: left-red/right-green divided by buy/sell ratio.
  • 分界线=Delta：分界线越偏右=买方越强；越偏左=卖方越强；居中=平衡。 Divider right = buyers dominate; left = sellers dominate; center = balanced.
  • 颜色语义统一：红=多头/上涨，绿=空头/下跌，与涨跌色一致。 Unified color semantics: red=bullish, green=bearish.
  • 无需第三维：分布与方向全部编码在2D图形内部，保持时间-价格坐标。 No 3rd dimension; all encoded inside the 2D element.
  • 可扩展：价格层分布可替换为真实逐笔/订单流数据。 Replaceable with real tick / order-flow data.

04 与传统K线对比 vs Candlestick / 对比

  • K线：仅OHLC；同OHLC无法区分内部结构。 OHLC only; identical OHLC hides internal structure.
  • 糖胡芦图：OHLC相同也能看出成交重心、价值区、多空主动权。 Reveals volume node, value area, aggression even with same OHLC.
  • K线+量柱：量柱是总量，不分价位不分买卖方。 Volume bar = total only, no price split, no side split.
  • 糖胡芦图：按价格层分布并按主动方拆分，三维信息压进2D。 Per-level distribution + aggressor split in 2D.
  • Market Profile：仅分布，不含方向。 Profile shows distribution, not direction.
  • 糖胡芦图：分布+方向融合体，可一键切回K线视图。 Fuses distribution + direction; toggle to candlesticks anytime.
